# 06. 开发执行任务清单

## 执行原则

每个任务必须小而清晰。不要一次性尝试“做一个完整暗黑类游戏”。

每次任务都要包含：

- 目标
- 修改范围
- 验收标准
- 测试方式
- 文档同步要求

## Task 001：初始化 Godot 原型工程

### 目标

在 `game/` 下初始化一个可打开的 Godot 4.x 2D 项目。

### 修改范围

- `game/project.godot`
- `game/scenes/main/`
- `game/scripts/core/`

### 验收标准

- Godot 可以打开项目。
- 运行后进入空白测试场景。
- 场景中存在基础 Camera2D。
- 项目名称显示为 `Stardust ARPG`。

### 测试方式

用 Godot 4.x 打开 `game/project.godot`，点击运行，确认进入 Main 场景。

## Task 002：实现玩家点地移动

### 目标

实现鼠标左键点击地面，玩家移动到目标点。

### 修改范围

- `game/scenes/player/`
- `game/scripts/player/`
- `game/scenes/maps/`

### 验收标准

- 玩家可以跟随鼠标点击位置移动。
- 到达目标点后停止。
- 移动速度可在配置中调整。
- 代码拆分出 MovementController，不把逻辑全部写进 Player 主脚本。

## Task 003：实现基础怪物 AI

### 目标

实现 1 种怪物的基础状态机：Idle / Chase / Attack / Dead。

### 验收标准

- 怪物能在玩家进入警戒范围后追击。
- 进入攻击距离后停止移动并攻击。
- 怪物血量归零后死亡。

## Task 004：实现普通攻击与伤害结算

### 目标

玩家可以点击怪物进行普通攻击。

### 验收标准

- 点击怪物后，玩家靠近怪物。
- 进入攻击范围后进行攻击。
- 怪物扣血并显示伤害数字。
- 怪物死亡后播放死亡逻辑。

## Task 005：实现掉落系统

### 目标

怪物死亡后根据掉落表生成装备或金币。

### 验收标准

- 怪物死亡后有概率掉落物品。
- 掉落物显示在地面。
- 玩家靠近或点击后可以拾取。
- 掉落规则来自数据文件，不硬编码在怪物脚本中。

## Task 006：实现背包与装备栏

### 目标

玩家可以查看背包、穿戴装备，并刷新属性。

### 验收标准

- 按快捷键打开背包。
- 背包中显示拾取物品。
- 点击装备可穿戴。
- 穿戴武器后玩家攻击力变化。

## Task 007：实现技能系统第一版

### 目标

实现普通攻击、冲刺斩、星尘爆裂。

### 验收标准

- 技能可以绑定快捷键。
- 技能有冷却。
- 技能可以造成伤害。
- 星尘爆裂具备范围伤害。

## Task 008：实现第一张野外测试图

### 目标

制作一张简单等距测试地图，包含阻挡、出生点、怪物刷新点、地牢入口。

### 验收标准

- 玩家不能穿墙。
- 怪物刷新在合理位置。
- 地牢入口可以触发场景切换。

## Task 009：实现地牢与 Boss 房

### 目标

实现 3~5 个房间的小地牢和一个 Boss 房。

### 验收标准

- 玩家可以从野外进入地牢。
- 地牢中有怪物群。
- Boss 房有单独 Boss。
- Boss 死亡后掉落稀有装备。

## Task 010：完成 15 分钟垂直切片

### 目标

整合营地、野外、地牢、Boss、掉落、背包、技能，形成第一版可玩 Demo。

### 验收标准

- 从运行游戏开始，玩家能完整体验一次：营地 → 野外 → 击杀 → 掉装 → 地牢 → Boss → 回城。
- 没有阻断流程的崩溃问题。
- README 更新运行方式。
